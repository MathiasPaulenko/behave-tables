# Security Policy

## Supported Versions

| Version | Supported |
| :--- | :--- |
| 1.x    | Yes      |
| < 1.0  | No       |

## Reporting a Vulnerability

If you discover a security vulnerability in behave-tables, please report it responsibly.

1. **Do not** open a public GitHub issue.
2. Email **mathias@paulenko.dev** with a description of the vulnerability and steps to reproduce.
3. You will receive an acknowledgment within 48 hours.
4. We will investigate and, if confirmed, release a fix as soon as possible.

## Disclosure

We follow coordinated disclosure. Once a fix is released, we will publish a GitHub Security Advisory crediting the reporter (unless they prefer to remain anonymous).
